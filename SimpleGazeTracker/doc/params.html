Parameters¶

1. Initial setup¶

At the first launch of SimpleGazeTracker, following parameters would probably be adjusted in accordance with your camera settings. In addition to these parameters, some camera-specific parameters may also be adjusted (see Camera specific parameters).

CAMERA_WIDTH, CAMERA_HEIGHT

These parameters must be equal to image resolution of the camera. Note that image resolution do not always correspond to sensor resolution. For example, if sensor resolution of the camera is 1280x960 and the camera is configured to do 2x2 binning (that is, merge 2x2 pixels into 1 pixel to increase sensitivity), image resolution is 640x480.

PREVIEW_WIDTH, PREVIEW_HEIGHT

These parameters adjusts preview pane on the application window. Basically these values should be equal to CAMERA_WIDTH and CAMERA_HEIGHT, you may want to scale up or down preview image size when resolution of the camera is too small or too large. For example, if resolution of the camera is 320x240, 640x480 would work fine.

ROI_WIDTH, ROI_HEIGHT

It is reccomened to set the same values as CAMERA_WIDTH and CAMERA_HEIGHT.

If field of view (FOV) of the camera is so wide that non-relevant objects are in the image, these parameters would be useful to make SimpleGazeTracker search pupil center region of the camera image. For example, image resolution of the camera is 640x480 and these parameters are set to 320x240, SimpleGazeTracker searches pupil only in the center region of 320x240 size of the image. This increases speed of detecting pupil and decreases rate of detection error. If the camera has crop mode, it is the better choice than using these parameters.

2. Parameters¶

General parameters¶

Most of parameters in this category concern gaze detection.

Parameter

Description

THRESHOLD

This parameter corresponds to PupilThreshold. See Camera setup guide for detal.

MAX_PUPIL_WIDTH

This parameter corresponds to MaxPupilWidth. See Camera setup guide for detal.

MIN_PUPIL_WIDTH

This parameter corresponds to MinPupilWidth. See Camera setup guide for detal.

PURKINJE_THRESHOLD

This parameter corresponds to PurkinjeThreshold. See Camera setup guide for detal.

PURKINJE_SEARCHAREA

This parameter corresponds to PurkinjeSearchArea. See Camera setup guide for detal.

PURKINJE_EXCLUDEAREA

This parameter corresponds to PurkinjeExcludeArea. See Camera setup guide for detal.

BINOCULAR

This parameter must be 0 or 1. If the value is 1, SimpleGazeTracker runs at binocular recording mode. Otherwise, it runs at monocular recording mode.

SHOW_DETECTIONERROR_MSG

This parameter must be 0 or 1. If the value is 1, Error message is overlaid on the camer apreview image if detection of pupil and the first Purkinje image is failed.

MORPH_TRANS

Apply morphological transformation if this value is greater than 1 (closing) or smaller than -1 (opening). Absolute value of this parameter corresponds to kernel size of the transformation.

Image parameters¶

Prameters in this category mainly concern dimension of camera image. Some cameras have specific parameters working with these parameters.

Parameter

Description

CAMERA_WIDTH

Width of the image sent from the camear.

CAMERA_HEIGHT

Height of the image sent from the camear.

PREVIEW_WIDTH

Width of the preview image on the screen of the Recorder PC.

PREVIEW_HEIGHT

Height of the preview image on the screen of the Recorder PC.

ROI_WIDTH

SimpleGazeTracker searches pupil and the first Purkinje image only from the center region specified by this parameter and ROI_HEIGHT.

ROI_HEIGHT

SimpleGazeTracker searches pupil and the first Purkinje image only from the center region specified by this parameter and ROI_WIDTH.

I/O parameters¶

Parameters for TCP/IP ports, USB I/O devices and data file format are in this category.

Parameter

Description

PORT_SEND

TCP/IP Port number for sending information to the Recorder PC. Default value is 10001.

PORT_RECV

TCP/IP Port number for receiving commands from the Recorder PC. Default value is 10000.

DELAY_CORRECTION

If delay in measuring gaze position is unacceptably long, you can use this parameter to correct delay. Unit of the value is millisedonds. Currently, only integer (not float) value is accepted. Default value is 0. See online document <http://gazeparser.sourceforge.net/adv/stim2capdelay.html> for detail.

OUTPUT_PUPILSIZE

This parameter must be 0 or 1. If the value is 1, pupil size (area of ellipse fitted to the pupil) is output to the data file.

USBIO_BOARD

ID or name of USB-IO device. See online document <http://gazeparser.sourceforge.net/hrd/usbio.html> for detail.

USBIO_AD

Port and range of analog input channel(s). See online document <http://gazeparser.sourceforge.net/hrd/usbio.html> for detail.

USBIO_DI

Port of digital input. See online document <http://gazeparser.sourceforge.net/hrd/usbio.html> for detail.

Camera specific parameters¶

Following parameters are camera-specific.

OpenCV edition¶

Parameter

Description

CAMERA_ID

This integer is passed to the constructor or CV::VideoCapture. Usually you need not write this parameter to the configuration file if you connect only one camera to your PC. If you have multiple cameras on your PC and SimpleGazeTracker does not use desirable camera, use this parameter to tell SimpleGazeTracker which camera should be used.

FRAME_RATE

Set this value to CV_CAP_PROP_FPS using cv::VideoCapture::set. Frame rate of the camera is set to this value if it is configurable from cv::VideoCapture::set.

EXPOSURE

Set this value to CV_CAP_PROP_EXPOSURE using cv::VideoCapture::set. Exposure of the camera is set to this value if it is configurable from cv::VideoCapture::set.

BRIGHTNESS

Set this value to CV_CAP_PROP_BRIGHTNESS using cv::VideoCapture::set. Brightness of the camera is set to this value if it is configurable from cv::VideoCapture::set.

CONTRAST

Set this value to CV_CAP_PROP_CONTRAST using cv::VideoCapture::set. Contrast of the camera is set to this value if it is configurable from cv::VideoCapture::set.

GAIN

Set this value to CV_CAP_PROP_GAIN using cv::VideoCapture::set. Gain of the camera is set to this value if it is configurable from cv::VideoCapture::set.

Parameter

Description

FlyCapture2 edition¶

Parameter

Description

CAMERA_N

Set camera ID (0=first device).

OFFSET_X

Set horizontal offset for cropping. Size of cropped area is specified by CAMERA_WIDTH and CAMERA_HEIGHT.

OFFSET_Y

Set vertical offset for cropping. Size of cropped area is specified by CAMERA_WIDTH and CAMERA_HEIGHT.

FRAME_RATE

Frame rate of the camera.

SHUTTER

Shutter duration.

CAMERA_MODE

Set camera mode. Only 0 (mode0) and 1 (mode1) are supported. SimpleGazeTracker supports only Format7 mode and RAW8 pixcel format.

BLUR_FILTER_SIZE

Set size of Gaussian filter that is applied to camera image. If the value is smaller than 2, Filter is not applied.
